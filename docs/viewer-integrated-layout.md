# Viewer 日历与教师列表整合布局

状态：已完成
确认日期：2026-07-11
更新日期：2026-07-12

## 目标

套磁日历是教师信息工作台的一部分，不再是与教师列表互斥的独立视图。日历固定在教师列表筛选栏上方；日历下方同时显示按推荐度排序的教师列表和教师详情。用户可以从高推荐教师开始处理套磁，也可以点击日历中的教师并在同一页面查看、编辑详情。

本布局不修改 ranking policy。联系状态已升级为 v4 并增加可选的 `约面试时间`；Viewer API 升级为 v4，同时移除了不再使用的整库保存、网页同步 Excel 和前端离线导入导出接口。

## 页面结构

```text
顶部标题与状态摘要
┌──────────────────────────────────────────────┐
│ 四周条状日历 + 独立学校/学院筛选 + 开合把手    │
│ 状态颜色与数量；点击日期后显示当天教师          │
└──────────────────────────────────────────────┘
教师列表筛选栏
┌──────────────────────────────┬───────────────┐
│ 教师列表                     │ 教师详情      │
│ 推荐等级、匹配分降序         │ 可收起/展开   │
│ 可直接修改套磁情况           │ 编辑联系记录  │
└──────────────────────────────┴───────────────┘
```

实现位于：

- `viewer/index.html`：固定的日历区、`.workbench`、教师列表和详情栏。
- `viewer/assets/app.js`：独立筛选、选择联动、侧栏状态、渐进表格和日历渲染。
- `viewer/assets/app.css`：桌面双列、窄屏内部滚动、面试标记和折叠样式。

## 统一状态与渲染

- 页面不再使用 Tab 在教师列表和日历之间切换。
- 已删除 `state.viewMode`、`renderViewTabs()`、`setViewMode()` 和 `renderActiveView()`。
- `applyCalendarFilters()` 生成 `state.calendarFiltered`；`applyFilters()` 单独生成教师列表的 `state.filtered`。
- 教师列表筛选不重建日历，日历学校或学院变化也不改变列表结果。
- `selectedKey` 是日历、列表和详情的唯一选择状态。
- 点击日历事件或教师行都会同步更新日历选中态、表格选中行和详情内容。
- 点击日期只更新当天教师区域；点击教师后再更新详情，移动端会滚动到详情。

## 筛选与排序

- 日历只受四周范围以及日历自己的学校、学院选项影响；学院选项随所选学校联动。
- 文本搜索、推荐等级、套磁情况、低证据、“查看已套磁”和“隐藏已标记”只作用于教师列表。
- 最低分筛选已删除。
- “查看已套磁”严格筛选 `套磁情况 === 已套磁`。
- “隐藏已标记”排除所有非空套磁情况。
- 同时启用“查看已套磁”和“隐藏已标记”时结果为空，符合两个筛选的交集语义。
- 教师列表先按 `强烈建议`、`可以考虑`、`暂不优先` 排序，再按匹配分降序。
- 日历为具有合法 `套磁时间` 或 `约面试时间` 的当前范围记录创建统计和下钻入口。
- “待补套磁日期”只列出 `套磁情况=已套磁` 且日期为空的记录。
- 从日历点击不在当前列表结果中的教师时，仍可在右侧显示其详情。

## 四周条状日历与回复情况

- 日历固定展示连续28天，每周一行；星期一至星期日是独立列头，行名以本周为基准显示“前2周、上周、本周、下周”等相对名称。
- 可用“上一周、本周、下一周”切换范围，每次移动一周。
- 每个日期只显示各回复状态的颜色和数量，不直接堆叠教师姓名。
- 顶部图例汇总当前四周的 `已发`、`官回`、`添加微信`、`约面试`、`考核`、`已满` 和 `未记录` 数量。
- 同一教师存在多个回复进展时会分别计入对应状态；点击日期后保留完整回复链。
- 日期选中态和今天使用额外描边，不覆盖回复状态颜色。
- 当天教师以横向卡片展示姓名、学校、学院和回复链；点击卡片联动教师详情。
- 选择“约面试”后可记录精确到分钟的 `约面试时间`；日期格和图例使用紫色菱形标记，教师卡片显示具体时间。
- 不再计算或展示同日同学院、短期同校发送频率警告。

## 日历折叠

- 日历底部中央使用高对比度、带文字的向下凸出把手；展开时显示“收起日历 ⌃”，收起后显示“展开日历 ⌄”。
- 收起后只保留四周范围和教师数量摘要，教师筛选与工作区自动获得更多高度。
- `localStorage` 只保存 `tutor-viewer-calendar-collapsed` 布尔偏好，恢复时保留当前日期逻辑。

## 详情栏折叠

- 大于 1100px 时，`.workbench` 使用 `minmax(600px, 1fr) minmax(490px, 610px)` 双列布局，详情栏比旧宽度增加约六分之一。
- 详情栏左边缘使用向外凸出的单一把手；展开时显示 `>` 并向右收起，收起后显示 `<` 并从右侧展开。
- 折叠会添加 `.sidebar-collapsed`，隐藏详情并让列表占满下方工作区；恢复时保留当前选中教师。
- `aria-controls` 和 `aria-expanded` 会随折叠状态更新，把手支持键盘操作。
- `localStorage` 只保存 `tutor-viewer-sidebar-collapsed` 布尔偏好，不保存教师 ID、联系状态或其他私人数据。
- 页面固定在一个视口内；详情栏和教师表格分别管理自身滚动，长证据不会拉高整个网页。
- 折叠过渡为 180ms，并尊重 `prefers-reduced-motion`。

## 响应式行为

- 大于 1100px：四周条带全宽；下方列表与详情双列，日历和详情分别可折叠。
- 901–1100px：四周条带全宽；列表和详情上下排列，隐藏详情把手但保留日历把手。
- 不超过 900px：保持相同四周结构，条带仅在日历区域内部横向滚动。
- 不超过 560px：日历筛选紧凑双列，图例横向滚动，详情编辑控件单列。
- 390px 视口下页面级 `scrollWidth` 不超过可视宽度；表格自身保留横向滚动。

## 不变约束

- `outputs/contact_status.json` 仍是唯一人工联系状态源。
- Excel 仍是查看和交付产物，不增加第二份前端状态库。
- 前端只展示工作簿中的 ranking policy 结论，不重新评分。
- `套磁时间` 继续表示实际套磁日期，不是未来计划发送日期。
- 点击日历只改变当前选中教师，不自动改变套磁情况、回复情况或日期。
- Viewer 继续只允许 loopback 服务访问，写请求沿用现有会话令牌和同源校验。

## 加载与操作性能

- `/api/data` 只返回教师摘要和联系状态，证据明细通过 `/api/detail?key=...` 按教师加载。
- 服务端按工作簿签名保存摘要缓存，并在内存中缓存详情 sheet 索引；工作簿变化会自动失效。
- 表格首批渲染 120 行，接近滚动底部时再追加。
- 搜索框使用 160 ms 防抖；选择教师只更新选中态和详情。
- 优化前后基线记录保存在忽略的本地项目记忆中；摘要冷启动、预热响应和单教师详情加载均已完成实机复测，详见 [固定视口与性能优化记录](viewer-ux-performance-todo.md)。
