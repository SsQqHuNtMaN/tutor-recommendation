# Viewer 日历与教师列表整合布局 TODO

状态：待实施  
确认日期：2026-07-11

## 目标

把套磁日历从独立视图改为教师信息工作台的一部分：日历位于教师列表和详情侧栏上方；日历下方保留按推荐度排序的教师列表和可折叠详情侧栏。用户可以从高推荐教师开始处理套磁，也可以点击日历中的教师并在同一页面查看、编辑详情。

本 TODO 只调整 Viewer 的信息架构和交互布局，不修改 ranking policy、工作簿 schema、联系状态 schema 或后端 API。

## 已确认的目标结构

```text
顶部标题与文件操作
筛选栏（所有区域共享）
┌──────────────────────────────────────────────┐
│ 套磁月历 / 移动端议程                        │
│ 回复情况图例、月份导航、密集发送提醒          │
└──────────────────────────────────────────────┘
┌──────────────────────────────┬───────────────┐
│ 教师列表                     │ 教师详情      │
│ 默认按推荐等级、匹配分排序   │ 可收起/展开   │
│ 可直接修改套磁情况           │ 编辑联系记录  │
└──────────────────────────────┴───────────────┘
```

核心交互：

- 页面不再用 Tab 在教师列表和日历之间切换，两者同时存在。
- 日历横跨教师列表与详情侧栏的总宽度。
- 教师列表和日历复用同一套学校、学院、推荐等级、分数、套磁情况与文本筛选。
- 点击日历事件或教师行都更新同一个 `selectedKey`，详情侧栏显示同一位教师。
- 详情侧栏可以收起；收起后教师列表占满下方工作区宽度。
- 桌面显示七列月历，窄屏继续显示现有日期议程。
- 日历事件颜色表达 `回复情况`，而不是 `套磁情况`。

## 不变约束

- `outputs/contact_status.json` 仍是唯一人工联系状态源。
- Excel 仍是查看和交付产物，不增加第二份前端状态库。
- 前端只展示工作簿中的 ranking policy 结论，不重新评分。
- `套磁时间` 继续表示实际套磁日期；本次布局改造不新增“计划发送日期”。
- 点击日历只改变当前选中教师，不自动改变套磁情况、回复情况或日期。
- Viewer 继续只允许 loopback 服务访问，写请求沿用现有会话令牌和同源校验。

## 实施清单

### 1. 重排 HTML 结构

- [ ] 在 `viewer/index.html` 中移除 `viewTabs` 导航。
- [ ] 把 `calendarPane` 从当前 `.content` 网格中移到筛选栏之后。
- [ ] 将日历下方区域命名为 `.workbench`，内部保留 `.table-pane` 和 `.detail-pane`。
- [ ] 在详情栏头部加入“收起”按钮，使用明确的 `aria-label`。
- [ ] 在侧栏收起时提供固定的“展开教师详情”按钮，避免只能通过刷新恢复。
- [ ] 保留日历导航、回复图例、提醒、月历、移动议程和待补日期区的现有 DOM 标识，减少逻辑迁移风险。

建议结构：

```html
<section class="calendar-pane">...</section>
<section class="workbench">
  <div class="table-pane">...</div>
  <aside class="detail-pane">...</aside>
  <button class="detail-expand" hidden>展开教师详情</button>
</section>
```

### 2. 简化视图状态

- [ ] 删除 `state.viewMode`、`renderViewTabs()`、`setViewMode()` 和 `renderActiveView()`。
- [ ] 数据加载或筛选后固定依次调用 `renderTable()`、`renderCalendar()`、`renderDetail()`。
- [ ] 保持 `selectedKey` 为日历、列表和详情的唯一选择状态。
- [ ] 增加 `state.sidebarCollapsed`，只控制布局，不影响选择或筛选结果。
- [ ] 可用 `localStorage` 只保存侧栏折叠布尔值；不得写入教师 ID、联系状态或其他私人数据。

### 3. 统一筛选与排序

- [ ] `applyFilters()` 只生成一次 `state.filtered`，供日历和教师列表共同消费。
- [ ] “查看已套磁”继续严格筛选 `套磁情况 === 已套磁`。
- [ ] “隐藏已标记”继续排除所有非空套磁情况。
- [ ] 教师列表继续使用 `compareRecords()`：先按 `强烈建议`、`可以考虑`、`暂不优先`，再按匹配分降序。
- [ ] 日历只为具有合法 `套磁时间` 的当前筛选记录创建事件。
- [ ] “待补套磁日期”只列出 `套磁情况=已套磁` 且日期为空的记录。
- [ ] 筛选导致选中教师消失时，选中当前结果中的第一位；无结果时清空详情。

### 4. 日历与详情联动

- [ ] 保留 `selectCalendarRecord(key)`，但不再切换视图。
- [ ] 点击日历事件后更新选中态并刷新表格行和详情栏。
- [ ] 若侧栏处于收起状态，点击日历事件不强制展开；“展开教师详情”按钮应显示已有选择。
- [ ] 桌面端点击事件后保持页面位置稳定，不自动滚动到页面底部。
- [ ] 移动端点击议程事件后滚动到详情区域，并确保详情在列表之后可访问。
- [ ] 列表选中项与日历选中事件使用一致但不覆盖回复颜色的描边样式。

### 5. 侧栏折叠布局

- [ ] `.workbench` 默认使用 `minmax(680px, 1fr) minmax(420px, 520px)` 两列布局。
- [ ] 收起时添加 `.sidebar-collapsed`，切换为单列并隐藏 `.detail-pane`。
- [ ] 展开/收起按钮更新 `aria-expanded`，键盘操作与鼠标操作一致。
- [ ] 侧栏展开时限制自身滚动，不让长证据详情无限拉高整个页面。
- [ ] 表格横向滚动保持在 `.table-pane` 内，不造成页面级横向溢出。
- [ ] 侧栏折叠动画控制在 150–200ms，并尊重 `prefers-reduced-motion`。

### 6. 响应式规则

- [ ] 宽度大于 1100px：月历全宽；下方列表与侧栏双列。
- [ ] 901–1100px：月历全宽；下方列表和详情改为上下排列，避免两列过窄。
- [ ] 不超过 900px：隐藏七列月历，显示现有日期议程。
- [ ] 不超过 560px：筛选器单列；日历工具栏换行；详情编辑控件单列。
- [ ] 移动端不显示无意义的侧栏折叠按钮，因为详情已经位于正常文档流中。
- [ ] 继续验证 390px 视口无页面级横向溢出。

### 7. 回复情况视觉规则

- [ ] 回复主色按进展优先级选择：`已满 > 考核 > 约面试 > 添加微信 > 官回 > 已发`。
- [ ] 多个回复同时存在时，月历紧凑事件显示最高进展和数量，例如 `添加微信 +2`。
- [ ] 移动议程显示完整回复链，例如 `已发 · 官回 · 添加微信`。
- [ ] 未填写回复情况的日历事件显示灰色“未记录”。
- [ ] 颜色之外始终保留文字，不以颜色作为唯一状态表达。
- [ ] 选中态使用额外描边或阴影，不覆盖回复状态的底色和文字。

## 推荐实施顺序

1. 只重排 DOM 和 CSS，使日历、列表、详情同时可见，暂不删除旧视图函数。
2. 接通日历、列表与详情的共同选择状态，完成桌面端联动。
3. 加入侧栏折叠状态与无障碍属性。
4. 删除不再使用的 Tab 和 `viewMode` 分支。
5. 调整移动端议程与详情滚动行为。
6. 完成自动化与人工验收后，再更新 README、runbook 和本 TODO 的状态。

- [ ] 日历位于列表与详情栏共同上方，并横跨两者宽度。
- [ ] 列表默认最先展示高推荐度教师。

- [ ] 收起详情栏后列表占满下方宽度，展开后恢复原选择。
- [ ] 学校、学院、推荐等级、套磁状态和“查看已套磁”同时影响列表与日历。

- [ ] 修改套磁时间或回复情况后，日历无需刷新页面即可更新。
